# Contributing

Thanks for contributing to `open-translation`.

## Audience Split

This repo keeps audience-specific documentation separate:

- [docs/end-users/README.md](../end-users/README.md) for end-user expectations
- [docs/contributors/README.md](./README.md) for contributor workflow, eval, and model assumptions

## Public Vs Private Material

Public repo content:

- `AGENTS.md`
- `docs/contributors/CONTRIBUTING.md`
- `docs/contributors/README.md`
- `docs/end-users/README.md`
- `LICENSE`
- `README.md`
- `skills/`

Local-only content:

- `.local/`

Do not add `.local/` files to git. Contributors should not depend on local-only planning or eval notes to understand the public repo.

Contributor workflow lives in [docs/contributors/README.md](./README.md).

## Pull Request Notes

- Explain the problem being fixed.
- Explain why the change belongs in that skill.
- Mention any overlap or terminology tradeoffs.
- Keep PRs small when possible.
